# Опис стратегії

- **Main branch містить стабільний, робочий код. Прямі коміти заборонені;**``
- **Для кожного завдання `Issue` створюється окрема гілка `feature branch` від main;**
- **Наступний формат назв: feature/issue-number-short-description `(наприклад, feature/15-auth-login)`;**
- **Після завершення роботи створюється `pull request` до основної гілки `main`;**
- **Інший учасник команди `(Астаф’єв перевіряє код Петренка, або Пилипчука)` переглядає зміни `Code Review`;**
- **Після затвердження зміни вливаються в `main`, а гілка видаляється.**
